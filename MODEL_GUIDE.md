## 1. 项目概述

### 1.1 这是什么？

**蛋白质判别系统** 是一个基于深度学习的蛋白质结构自动分类工具。你给它一个 PDB 格式的蛋白质结构文件，它就能告诉你这个蛋白质属于四大类别中的哪一类，并给出置信度。

### 1.2 四大分类

| 编号 | 中文名 | 英文名 | 含义 | 例子 |
|------|--------|--------|------|------|
| 0 | **酶/受体** | Protein-Ligand | 蛋白质与小分子配体结合 | 药物靶点蛋白、酶 |
| 1 | **蛋白质-蛋白质复合物** | Protein-Protein | 两个蛋白质相互结合 | 抗体-抗原复合物 |
| 2 | **蛋白质-核酸复合物** | Protein-Nucleic Acid | 蛋白质与 DNA/RNA 结合 | 转录因子、核糖体蛋白 |
| 3 | **核酸-配体复合物** | Nucleic Acid-Ligand | 核酸与小分子结合 | RNA 适配体、核酶 |

### 1.3 为什么重要？

- **药物研发**：大多数药物靶点都是酶或受体（类别0），快速识别蛋白质类型有助于药物筛选
- **结构生物学**：蛋白质结构数据库中数万条记录，手动分类不现实
- **功能预测**：知道蛋白质的"社会关系"（和谁结合）是推断其功能的关键线索

### 1.4 输入与输出

```
输入: 一个 .pdb 文件 (蛋白质三维结构)
  ↓
[特征提取] → [图神经网络] → [分类器]
  ↓
输出: {
  "name": "结合蛋白_1a07",
  "category": "酶/受体",
  "confidence": 0.934,  ← 模型预测置信度 (0~1)
  "all_probabilities": {
    "酶/受体": 0.934,
    "蛋白质-蛋白质复合物": 0.046,
    "蛋白质-核酸复合物": 0.012,
    "核酸-配体复合物": 0.008
  }
}
```

> **注意**: 数据库中的蛋白质分两种来源 — ① 数据集导入的已知分类记录（`confidence=NULL`，前端显示"已知分类"标签）；② 通过 `/predict` API 上传预测的记录（`confidence` 为实际模型置信度，前端显示百分比）。不要把 NULL 误解为"模型100%确信"。

## 2. 背景知识

> 本节为零基础读者准备。如果你已经了解蛋白质结构和图神经网络，可以跳过。

### 2.1 PDB 文件格式

PDB（Protein Data Bank）是存储蛋白质三维结构的标准格式。一个 PDB 文件看起来像这样：

```
ATOM      1  N   ALA A   1       0.000   0.000   0.000  1.00  0.00           N
ATOM      2  CA  ALA A   1       1.458   0.000   0.000  1.00  0.00           C
ATOM      3  C   ALA A   1       2.009   1.420   0.000  1.00  0.00           C
...
```

关键信息：
- **ATOM** 行：每个原子的三维坐标
- **残基名**（如 ALA）：氨基酸的三字母缩写
- **链 ID**（如 A）：标识不同的蛋白质链
- **CA 原子**：α-碳原子，位于氨基酸骨架，是表示蛋白质结构的"关键点"

### 2.2 什么是图神经网络 (GNN)？

传统神经网络处理的是**固定大小**的数据（如图片是 224×224 像素的网格）。但蛋白质分子是**不规则**的三维结构——不同蛋白质有不同数量的原子，排列也不规则。

图神经网络解决了这个问题：
- **节点 (Node)** = 每个氨基酸残基（如第5个丙氨酸）
- **边 (Edge)** = 空间上相邻的残基之间连线（距离 < 8Å 的 CA 原子之间）

```
    残基1 —— 残基2
      |    \    |
      |     \   |
    残基3 —— 残基4
```

图神经网络通过**消息传递**（Message Passing）工作——每个节点收集邻居节点的信息来更新自己的特征，经过多轮传递后，每个节点"知道"了它周围的结构环境。

### 2.3 氨基酸与蛋白质

蛋白质由 20 种标准氨基酸组成，每种用一个大写字母表示：

| 字母 | 全称 | 中文名 | 特性 |
|------|------|--------|------|
| A | Alanine | 丙氨酸 | 疏水 |
| C | Cysteine | 半胱氨酸 | 可形成二硫键 |
| D | Aspartate | 天冬氨酸 | 酸性 |
| ... | ... | ... | ... |
| Y | Tyrosine | 酪氨酸 | 芳香族 |

氨基酸序列决定了蛋白质折叠成什么形状，而形状决定了它的功能。我们的模型同时利用了**序列信息**和**结构信息**。

## 4. 数据准备

### 4.1 数据来源

数据来自 **PDBbind 数据库** (v2016–v2020R1) 和 **PDBbind-PLANET** 数据集。

| 数据集 | 版本 | 描述 | P-L 样本 |
|--------|------|------|----------|
| PDBbind v2020R1 | 2020 | 蛋白质-配体结合亲和力基准 | ~19,037 |
| PDBbind v2019 | 2019 | 同上 | ~17,600 |
| PDBbind v2018 | 2018 | 同上 | ~16,100 |
| PDBbind v2017 | 2017 | 同上 | ~14,700 |
| PDBbind v2016 | 2016 | 同上 | ~13,300 |
| PDBbind-PLANET | 2020 | 结合亲和力预测训练/验证/测试划分 | ~19,400 |

另有少量 P-P（~91）、P-NA（~100）、NA-L（~30）样本来自 v2020R1。

合并去重后共有 **39,224 个唯一蛋白质结构**。

### 4.2 特征提取流程

从每个 PDB 文件提取的特征分为两大类：**全局特征**和**图结构特征**。

#### 全局特征 (433 维向量)

这是对整个蛋白质的"宏观"描述：

```
┌──────────────────────┬──────┬──────────────────────────┐
│ 特征类型              │ 维度  │ 描述                      │
├──────────────────────┼──────┼──────────────────────────┤
│ 氨基酸组成 (Composition)│  23  │ 20种标准AA + 3种非标AA的频率│
│ 物化性质 (Physicochem) │   4  │ 疏水性、电荷、极性、体积均值  │
│ 空间特征 (Spatial)     │   6  │ 回旋半径、距离统计、惯性张量  │
│ 二肽组成 (Dipeptide)   │ 400  │ 相邻氨基酸对的频率 (20×20)  │
├──────────────────────┼──────┼──────────────────────────┤
│ 合计                  │ 433  │                          │
└──────────────────────┴──────┴──────────────────────────┘
```

**关键概念解释**：

- **氨基酸组成**：Protein A 中丙氨酸占 10%，甘氨酸占 5%……这个比例反映蛋白质的整体氨基酸偏好
- **物化性质**：疏水性强的蛋白质倾向于埋在细胞膜中，带电荷多的蛋白质容易溶于水
- **回旋半径**：描述蛋白质"有多紧凑"——球形 vs 松散链状
- **惯性张量特征值**：描述蛋白质的三维形状（像椭球？像长棒？像扁平盘？）
- **二肽组成**：相邻的氨基酸对出现频率，捕捉局部序列模式（如"KR"常见于核定位信号）

#### 图结构特征

每个氨基酸残基是图的一个**节点**，节点特征是该氨基酸的 **one-hot 编码**（23 维，其中 20 维对应标准氨基酸，1 维为非标准氨基酸，1 维为核酸残基）。

节点之间的**边**基于 CA 原子在三维空间中的距离：

```python
if distance(CA_i, CA_j) < 8.0 Å:  # 8埃 = 0.8纳米
    连接节点 i 和 j
```

这意味着空间上靠近的氨基酸被连接起来，即使它们在序列上可能相隔很远。

### 4.3 类别均衡策略

由于 P-L 类别有超过 10 万个样本，而其他类别只有几十到一百个，直接训练会导致模型"偷懒"——只预测 P-L 就能拿到 95%+ 的准确率。

我们采用两个策略：

1. **类别级采样**：每个类别最多取 `pdb_samples_per_class=2000` 个样本
2. **加权损失函数**：少数类的分类错误被赋予更高的惩罚权重

```python
# 权重 = 总样本数 / (类别数 × 该类别样本数)
# P-L (1414训练样本): 权重 = 1561/(4×1414) = 0.28
# P-P (65训练样本):   权重 = 1561/(4×65)   = 6.00
# P-NA (62训练样本):  权重 = 1561/(4×62)   = 6.29
# NA-L (20训练样本):  权重 = 1561/(4×20)   = 19.51
```

这意味着错分一个 NA-L 样本的代价是错分一个 P-L 样本的 **70 倍**。

## 5. 模型架构

### 5.1 模型总览

```
         PDB 文件
             │
    ┌────────┴────────┐
    │   特征提取        │
    │  (NumPy 管道)     │
    └────────┬────────┘
             │
    ┌────────┴────────┐
    │  全局特征 433维   │        │  节点特征 [N, 23]  │
    │  (aa组成+物化     │        │  (氨基酸 one-hot)  │
    │   +空间+二肽)     │        │                     │
    └────────┬────────┘        └──────────┬──────────┘
             │                            │
             │              ┌─────────────┴─────────────┐
             │              │   邻接矩阵 [N, N]           │
             │              │   (CA距离 < 8Å → 边)       │
             │              └─────────────┬─────────────┘
             │                            │
    ┌────────▼────────┐          ┌────────▼────────┐
    │  Global Encoder │          │   Node Embedding │
    │  Linear(433→128)│          │  Linear(23→128)  │
    │  + ReLU + BN    │          └────────┬────────┘
    │  Linear(128→128)│                   │
    │  + ReLU         │          ┌────────▼────────┐
    └────────┬────────┘          │   3× GraphConv   │
             │                   │   (消息传递)      │
             │                   │   每层:           │
             │                   │   归一化邻接矩阵   │
             │                   │   × 节点特征      │
             │                   │   → Linear → BN  │
             │                   │   → ReLU → Drop  │
             │                   └────────┬────────┘
             │                            │
             │                   ┌────────▼────────┐
             │                   │  Global Pooling  │
             │                   │  Mean + Max      │
             │                   │  → [batch, 128]  │
             │                   └────────┬────────┘
             │                            │
             └──────────┬─────────────────┘
                        │
               ┌────────▼────────┐
               │  Concat [256]   │
               │  (graph_128 +   │
               │   global_128)   │
               └────────┬────────┘
                        │
               ┌────────▼────────┐
               │   Classifier    │
               │  Linear(256→128)│
               │  + ReLU + Drop  │
               │  Linear(128→64) │
               │  + ReLU + Drop  │
               │  Linear(64→4)   │
               └────────┬────────┘
                        │
                 ┌──────▼──────┐
                 │  Softmax    │
                 │  4类概率     │
                 └─────────────┘
```

### 5.2 核心组件详解

#### (A) 图卷积层 (GraphConvLayer)

这是 GNN 的核心运算。每一层执行以下步骤：

```
输入: 节点特征 X [batch, N, 128], 邻接矩阵 A [batch, N, N]

步骤1 — 归一化:  A_norm = A / degree(A)
  # 每个节点收集邻居信息时，求平均而非求和

步骤2 — 消息传递:  messages = A_norm @ X
  # 矩阵乘法：每个节点的"新特征" = 邻居节点特征的加权平均

步骤3 — 变换:  out = Linear(messages) → BatchNorm → ReLU → Dropout
  # 通过可学习的权重矩阵进行特征变换
```

**通俗理解**：每一层图卷积，节点都会"和邻居们聊一次天"。聊了三轮之后（3层卷积），每个节点就知道了它三跳范围内的结构信息。

#### (B) 全局编码器 (GlobalEncoder)

先将 433 维的全局特征压缩到 128 维：

```
433维 → Linear → 128维 → BN → ReLU → Dropout
       → Linear → 128维 → BN → ReLU
```

这是经典的全连接网络。两个线性层之间加了 BatchNorm（批归一化，稳定训练）和 ReLU（非线性激活）。

#### (C) 图池化 (Graph Pooling)

GNN 输出是每个节点的特征向量，但我们最终需要整个蛋白质的特征。池化操作把所有节点信息压缩为一个向量：

```python
graph_feat = X.mean(dim=1) + X.max(dim=1)[0]
# 平均池化: 提取"整体模式"
# 最大池化: 提取"显著特征"
# 两者相加: 既看全貌也看亮点
```

#### (D) 分类头 (Classifier)

融合后的 256 维特征（图特征128 + 全局特征128）经过三层全连接网络，最终输出 4 个值（logits），经 softmax 转为概率：

```python
logits = [2.3, -1.5, -0.8, -2.1]
probs = softmax(logits) = [0.85, 0.02, 0.04, 0.09]
prediction = argmax(probs) = 0  # 酶/受体
confidence = probs[0] = 0.85    # 85% 置信度
```

### 5.3 模型参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 总参数量 | 167,748 | 较小的模型 |
| 隐藏维度 | 128 | 中间特征的维度 |
| 图卷积层数 | 3 | 消息传递的轮数 |
| Dropout | 0.3 | 训练时随机关闭 30% 神经元，防止过拟合 |
| 激活函数 | ReLU | max(0, x)，引入非线性 |
| 优化器 | Adam | 自适应学习率优化算法 |

| 最大序列长度 | 1000 | 超过 1000 个残基的蛋白质会被截断 |

## 6. 训练过程

### 6.1 数据划分

```
全部 2230 个样本
    │
    ├── 训练集: 70% (1561 样本) — 用于学习模型参数
    ├── 验证集: 15% (334 样本)  — 用于调整超参数和早停
    └── 测试集: 15% (335 样本)  — 用于最终评估（模型从未见过）
```

各类别训练分布：P-L: 1414, P-P: 65, P-NA: 62, NA-L: 20

### 6.2 训练循环

一个 epoch（一轮训练）的流程：

```python
for epoch in range(100):
    # === 训练阶段 ===
    model.train()  # 启用 dropout 等
    for batch in train_loader:
        optimizer.zero_grad()       # 清空梯度
        outputs = model(batch)      # 前向传播
        loss = criterion(outputs, labels)  # 计算损失
        loss.backward()             # 反向传播（计算梯度）
        optimizer.step()            # 更新参数

    # === 验证阶段 ===
    model.eval()  # 禁用 dropout
    with torch.no_grad():
        val_loss, val_acc = evaluate(model, val_loader)

    scheduler.step(val_loss)  # 学习率调度

    if val_acc > best_val_acc:
        save_model()  # 保存最佳模型
```

### 6.3 训练曲线特征

```
Epoch  1: Train Loss 1.190, Val Acc 15.6%  ← 模型刚开始学习
Epoch 10: Train Loss 0.522, Val Acc 55.1%  ← 快速进步
Epoch 13: Val Acc 89.2% (最佳)              ← 达到峰值
Epoch 20: Train Loss 0.239, Val Acc 88.3%  ← 开始过拟合
Epoch 28: Early Stopping                    ← 自动停止
```

### 6.4 防止过拟合的措施

1. **Dropout (0.3)**：训练时随机丢弃 30% 的神经元，迫使网络不依赖特定节点
2. **Batch Normalization**：每层归一化输出，稳定训练过程
3. **Early Stopping (patience=15)**：验证准确率不再提升时停止，避免"死记硬背"训练集
4. **Weight Decay (1e-5)**：L2 正则化，惩罚过大的权重值
5. **ReduceLROnPlateau**：学习率自适应降低，防止在最优解附近震荡

| 蛋白质-蛋白质 (P-P) | **21.1%** | 4/19 | 仅65训练样本，特征与P-L相似 |
| 蛋白质-核酸 (P-NA) | **73.7%** | 14/19 | 核酸结合特征较明显 |
| 核酸-配体 (NA-L) | **80.0%** | 4/5 | 结构差异大，易区分但样本极少 |
| **总体** | **88.4%** | 296/335 | |

### 7.2 混淆矩阵分析

```
                     预测→
实际↓         P-L    P-P   P-NA  NA-L
P-L          274     12     5     1
P-P           10      4     5     0
P-NA           2      2    14     1
NA-L           0      1     0     4
```

**解读**：
- P-L 最容易被正确识别（274/292），少量被误判为 P-P（12）和 P-NA（5）
- P-P 最容易与 P-L 混淆（10/19 被误判为 P-L）——因为两者都是蛋白质为主的结构
- P-NA 和 NA-L 一旦被正确识别，准确率很高，但少数会被误判为主流类别

### 7.3 性能瓶颈分析

**P-P（蛋白质-蛋白质复合物）准确率低的原因**：

1. **训练样本极少**：仅 65 个训练样本，深度学习通常需要数百到数千样本
2. **特征相似性**：P-P 和 P-L 的全局特征分布高度重叠——都是蛋白质，基本组成相似
3. **图结构难以区分**：两个蛋白质的结合界面与蛋白质-小分子结合位点在局部图结构上可能相似

**改进方向**：
- 获取更多 P-P 数据（如 Docking Benchmark、BioLiP）
- 引入界面特征（interfacial residues、contact maps）
- 使用预训练模型（如 ProtBERT、ESM）提取更好的蛋白质表示

### 8.7 已知注意事项

1. **CONFIG 不允许有重复键** — Python dict 重复键取最后一个值，修改训练参数时确保不重复
2. **类别权重必须用 Python int** — 用 tensor 做 dict key 会导致 hash 不同，权重计算失效
3. **数据库 `confidence` 字段** — 数据集导入的记录应设为 `None`（而非 `1.0`），前端据此区分"已知分类"和模型预测
4. **NA-L 文件需要 `allow_na=True`** — 核酸链的解析需要特殊处理，`extract_features` 和 `ProteinPredictor.predict` 都要传递此参数

### 9.3 关键概念速查

| 概念 | 一句话解释 |
|------|-----------|
| **Epoch** | 把所有训练数据完整过一遍 = 1 个 epoch |
| **Batch** | 每次更新参数时使用的一小批样本 |
| **Learning Rate** | 参数更新的步长——太大则震荡，太小则太慢 |
| **Overfitting** | 过拟合：模型记住了训练集的"答案"，但不会做新题 |
| **Dropout** | 训练时随机"关闭"一些神经元，防止它们形成依赖 |
| **Softmax** | 把任意分数转为概率分布（和为1） |
| **Cross-Entropy Loss** | 分类问题最常用的损失函数，衡量预测概率与真实标签的差距 |
| **One-Hot Encoding** | 把类别转成向量：[1,0,0,0] = 类别0, [0,1,0,0] = 类别1 |

## 10. 应用前景与局限性

### 10.1 应用场景

1. **药物研发管线加速**
   - 高通量筛选初期：快速识别靶点蛋白质类型
   - 虚拟筛选前：确认目标蛋白属于酶/受体类（可药性最高）

2. **结构基因组学**
   - 批量处理 PDB 新释放的结构
   - 自动标注蛋白质的功能类别
   - 辅助蛋白质功能预测管道

3. **生物信息学教育**
   - 可视化蛋白质分类过程
   - 学生通过 Web 界面上传和分析蛋白质
   - 理解深度学习在结构生物学中的应用

4. **数据库管理**
   - 对自建蛋白质结构数据库进行自动分类
   - 蛋白质结构文件的自动标签和组织

### 10.2 当前局限

| 局限 | 说明 | 严重程度 |
|------|------|----------|
| 少数类精度低 | P-P 仅 21% 准确率，受限于训练数据量 | 高 |
| 特征工程依赖 | 使用手工设计的特征（433维），可能遗漏信息 | 中 |
| 只支持 PDB 格式 | 不支持 mmCIF 格式（新标准），.cif 文件需先转换 | 中 |
| 序列长度限制 | 超过 1000 残基的蛋白质会被截断 | 低 |
| 无分子动力学信息 | 只使用静态结构，不利用构象动态信息 | 低 |

### 10.3 未来改进方向

## 附录 B：常见问题 (FAQ)

**Q: 训练需要多长时间？**
A: 在普通 CPU 上约 20-30 分钟（主要耗时在特征提取），在 CUDA GPU 上约 10-15 分钟。

**Q: 为什么我的蛋白质总是被分类为"酶/受体"？**
A: P-L 是最大的类别。如果你的蛋白质确实属于其他类别但被误判，可能是因为该类别训练数据不足。可以尝试提供更多类似蛋白质的 PDB 文件。

**Q: 如何处理 .cif 格式的文件？**
A: 虽然 API 接受 .cif 文件，但解析功能有限。建议先用 PyMOL 或 OpenBabel 将 .cif 转换为 .pdb 格式。

**Q: 如何添加新的分类？**
A: 参见 `CLAUDE.md` 中的"添加新分类"步骤，涉及修改 5 个文件并重新训练模型。

**Q: 模型文件存储在哪里？**
A: `protein_classifier_project/backend/model/protein_classifier.pth`，约 700KB。
